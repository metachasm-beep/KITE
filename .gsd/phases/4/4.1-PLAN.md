# Plan 4.1: Prisma Schema & Database Setup

## Objective
Initialize Prisma ORM and define the full transactional schema for VOIDLAB covering Users, Addresses, Carts, Cart Items, Orders, and Order Items.

## Context
- .gsd/ARCHITECTURE.md (Database Models section)
- Prisma schema draft referenced from `Prisma schema draft for Postgres.docx`
- NextAuth session already provides user identity. Prisma will store transactional state.

## Tasks

<task type="auto">
  <name>Install Prisma</name>
  <files>package.json</files>
  <action>
    1. Install `prisma` and `@prisma/client`.
    2. Initialize Prisma with `npx prisma init --datasource-provider postgresql`.
  </action>
  <verify>Check `prisma/schema.prisma` exists and `@prisma/client` is in package.json.</verify>
  <done>Prisma is initialized.</done>
</task>

<task type="auto">
  <name>Author the Full Transactional Schema</name>
  <files>prisma/schema.prisma</files>
  <action>
    Define the following Prisma models tightly aligned to VOIDLAB requirements:
    - **User**: id, email, name, role (`CUSTOMER` | `ADMIN`), image, createdAt. Provider accounts linked via NextAuth.
    - **Account**: id, userId, provider, providerAccountId (for NextAuth Prisma Adapter compatibility).
    - **Address**: id, userId, line1, city, state, postalCode, country.
    - **Cart**: id, userId, items, createdAt.
    - **CartItem**: id, cartId, artifactId (String, maps to CMS slug), quantity, price.
    - **Order**: id, userId, totalAmount, status (enum: `PENDING` | `PROCESSING` | `SHIPPED` | `DELIVERED` | `CANCELLED`), createdAt.
    - **OrderItem**: id, orderId, artifactId, title, quantity, price.
    - **OrderTimeline**: id, orderId, status, note, timestamp.
  </action>
  <verify>Schema parses with `npx prisma validate` without errors.</verify>
  <done>Schema is authored and validated.</done>
</task>

<task type="auto">
  <name>Create Prisma Client Singleton</name>
  <files>lib/db.ts</files>
  <action>
    1. Create `lib/db.ts` exporting a global Prisma client singleton preventing connection pool exhaustion in development hot reloads.
  </action>
  <verify>File exists and is importable.</verify>
  <done>Prisma client is available globally.</done>
</task>

## Success Criteria
- [ ] Prisma initialized and schema authored with 8 models.
- [ ] `lib/db.ts` singleton in place.
- [ ] Schema validates cleanly.
