# Plan 4.3: Cart & Order Workflow APIs

## Objective
Build the API layer for Cart management and Order creation, backed by Prisma. This establishes the commerce backbone connecting product selection to order confirmation.

## Context
- Depends on 4.1 (Prisma schema) being complete.
- Cart is session-aware — only auth'd users can add to cart.
- Order creation locks the Cart and creates an immutable purchase record.

## Tasks

<task type="auto">
  <name>Cart API Routes</name>
  <files>app/api/cart/route.ts</files>
  <action>
    1. Create `GET /api/cart` returning the current user's Cart and its CartItems from Prisma.
    2. Create `POST /api/cart` accepting `{ artifactId, quantity, price }` and upserting a CartItem onto the user's active Cart.
    3. Protect routes with `getServerSession` — return `401` for unauthenticated requests.
  </action>
  <verify>API routes respond correctly. POST creates/updates a CartItem in the DB.</verify>
  <done>Cart API is operational.</done>
</task>

<task type="auto">
  <name>Order Creation API Route</name>
  <files>app/api/orders/route.ts</files>
  <action>
    1. Create `POST /api/orders` accepting the user's current Cart.
    2. Atomically: create an Order record, copy CartItems as OrderItems, then clear the Cart.
    3. Create an initial OrderTimeline event: `{ status: "PENDING", note: "Order received" }`.
    4. Return the created Order ID to the client for confirmation navigation.
  </action>
  <verify>POST to `/api/orders` successfully creates an Order record in Prisma.</verify>
  <done>Order creation flow is complete.</done>
</task>

## Success Criteria
- [ ] Cart reads and writes work against PostgreSQL via Prisma.
- [ ] Order creation is atomic (no partial states).
- [ ] OrderTimeline entry is automatically seeded on order creation.
