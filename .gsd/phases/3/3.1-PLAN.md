# Plan 3.1: Bridge Layer Setup (CMS Mock)

## Objective
Establish the Data Bridge Layer that abstracts Headless CMS fetching from components. Before committing to a specific CMS (like Sanity/Payload), we will create a mocked API bridge that adheres to the strict VOIDLAB schemas.

## Context
- .gsd/ARCHITECTURE.md 
- Decouples UI from the CMS so the backend can be natively swapped out later.
