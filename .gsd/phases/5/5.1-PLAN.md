# Plan 5.1: GitHub Repository & Version Control Finalization

## Objective
Push the entire VOIDLAB codebase to a named private GitHub repository as the authoritative source of truth, ensuring the repo is clean and ready for CI/CD deployment from Render.

## Tasks

<task type="manual">
  <name>Create the GitHub Repository</name>
  <action>
    1. Go to https://github.com/new.
    2. Name it `KITE` (matching the local folder name).
    3. Set visibility to **Public** or **Private** based on preference.
    4. Do NOT initialize with a README (our repo already has commits).
    5. Click "Create repository" and copy the remote URL.
  </action>
  <done>Repository exists on GitHub with a remote URL.</done>
</task>

<task type="auto">
  <name>Create .gitignore</name>
  <files>.gitignore</files>
  <action>
    1. Ensure the `.gitignore` is comprehensive covering:
       - `node_modules/`
       - `.next/`
       - `.env`
       - `.env.local`
       - `prisma/dev.db`
       - `*.log`
  </action>
  <verify>`.gitignore` file is correct and sensitive files are excluded.</verify>
  <done>`.gitignore` is authored and committed.</done>
</task>

<task type="auto">
  <name>Add Environment Files to .gitignore</name>
  <files>.gitignore</files>
  <action>
    Verify `.env.local` is listed in `.gitignore`. It must NEVER be committed.
    Provide the `.env.local.example` file as a safe template reference.
  </action>
  <done>Secrets are protected from the repo.</done>
</task>

<task type="manual">
  <name>Link Remote and Push</name>
  <action>
    Run the following commands in the terminal:
    ```bash
    git remote add origin https://github.com/[YOUR_USERNAME]/KITE.git
    git branch -M main
    git push -u origin main
    ```
  </action>
  <done>All commits are visible on GitHub.</done>
</task>

## Success Criteria
- [ ] `.gitignore` excludes all secrets and build artifacts.
- [ ] GitHub repo exists and is linked as `origin`.
- [ ] All 4 phase commits are visible on GitHub.
