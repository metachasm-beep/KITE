# Plan 5.3: Render Web Service & Production Launch

## Objective
Connect the GitHub repository to a new Render Web Service, configure production environment variables, and verify the live deployment of the VOIDLAB storefront.

## Tasks

<task type="manual">
  <name>Configure Google OAuth Redirect URIs</name>
  <action>
    1. Go to Google Cloud Console > APIs & Services > Credentials.
    2. Edit your OAuth 2.0 Client ID.
    3. Under **Authorized redirect URIs**, add `https://your-app-name.onrender.com/api/auth/callback/google`.
    4. Save changes.
  </action>
  <done>Google OAuth will recognize the Render production URL.</done>
</task>

<task type="manual">
  <name>Create Render Web Service</name>
  <action>
    1. Go to Render Dashboard > New + > **Web Service**.
    2. Connect your GitHub account and select the `KITE` repository.
    3. Name the service `voidlab`.
    4. Runtime: **Node**.
    5. Build Command: `npm run build`.
    6. Start Command: `npm run start`.
  </action>
  <done>Web Service is configured to pull from GitHub.</done>
</task>

<task type="manual">
  <name>Set Production Environment Variables</name>
  <action>
    1. In the Render Web Service settings, go to the **Environment** tab.
    2. Click "Add Environment Variable" and add all keys from `.env.local.example`:
       - `GOOGLE_CLIENT_ID`
       - `GOOGLE_CLIENT_SECRET`
       - `NEXTAUTH_SECRET`
       - `NEXTAUTH_URL` (use the provided Render URL: `https://...onrender.com`)
       - `ADMIN_EMAIL`
       - `DATABASE_URL` (use the **Internal** Database URL if database is in the same Render region).
    3. Click **"Save Changes"**.
  </action>
  <done>Production environment matches the local logic.</done>
</task>

<task type="manual">
  <name>Deploy & Final Audit</name>
  <action>
    1. Trigger a manual deploy if it doesn't start automatically.
    2. Once "Live", visit the URL.
    3. Verify:
       - [ ] Homepage Hero and Philosophy sections render.
       - [ ] `/collections` shows the grid.
       - [ ] Google Sign-In works.
       - [ ] `/admin` is accessible only to the ADMIN_EMAIL user.
  </action>
  <done>VOIDLAB is officially live and secure.</done>
</task>

## Success Criteria
- [ ] Render build succeeds (`Next.js` optimized production build).
- [ ] App is reachable at the `*.onrender.com` subdomain.
- [ ] Transactional logic (Cart/Auth) persists in the Render Postgres DB.
- [ ] Admin dashboard is functional in production.
