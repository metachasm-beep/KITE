# Plan 5.2: Render PostgreSQL & Database Migration

## Objective
Provision a PostgreSQL database on Render, run the Prisma migration to apply the schema, and verify connectivity.

## Tasks

<task type="manual">
  <name>Create Render PostgreSQL Database</name>
  <action>
    1. Go to https://dashboard.render.com
    2. Click **"New +"** → **"PostgreSQL"**
    3. Name it `voidlab-db`
    4. Select the free plan (or lowest available tier)
    5. Click **"Create Database"**
    6. From the database page, copy the **"Internal Database URL"** (used by Render services) and the **"External Database URL"** (used for running migrations locally).
  </action>
  <done>Postgres instance is running on Render and connection URLs are available.</done>
</task>

<task type="auto">
  <name>Configure Local .env.local for Migration</name>
  <files>.env.local</files>
  <action>
    1. Create `.env.local` (if it doesn't exist) using `.env.local.example` as the template.
    2. Fill in `DATABASE_URL` with the **External Database URL** from Render for running migrations.
    3. Fill in remaining required vars:
       - `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` (from Google Cloud Console OAuth 2.0)
       - `NEXTAUTH_SECRET` (generate with `openssl rand -base64 32`)
       - `NEXTAUTH_URL=https://your-render-service.onrender.com` (once the service exists)
       - `ADMIN_EMAIL=your_email@gmail.com`
  </action>
  <done>.env.local is populated with all required secrets.</done>
</task>

<task type="auto">
  <name>Run Prisma Migration</name>
  <files>prisma/schema.prisma</files>
  <action>
    Run the following commands in the terminal once the DATABASE_URL is set:
    ```bash
    npx prisma migrate dev --name init
    npx prisma generate
    ```
    This will apply the schema to the Render Postgres DB and generate the Prisma Client.
  </action>
  <verify>Prisma studio (`npx prisma studio`) shows all 9 tables created.</verify>
  <done>Database is migrated and all models are live in Postgres.</done>
</task>

## Success Criteria
- [ ] Render PostgreSQL is provisioned and running.
- [ ] `.env.local` is fully configured with all secrets.
- [ ] `npx prisma migrate dev` completes without errors.
- [ ] All 9 tables are live in the Postgres database.
