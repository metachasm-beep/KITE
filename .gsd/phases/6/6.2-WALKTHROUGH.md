# Shiprocket Integration Walkthrough (Phase 6)

I have successfully integrated the Shiprocket API to automate order fulfillment and tracking for KITE.

## 1. Database & Schema Updates
- **Prisma Schema**: Updated the `Order` model to include Shiprocket tracking fields: `shiprocketOrderId`, `shipmentId`, `awbCode`, `courierName`, and `trackingStatus`.
- **Address Enhancements**: Added a `phone` field to the `Address` model and updated the `Order` model to store snapshots of `shippingAddress` and `billingAddress` (JSON) to ensure long-term audit compatibility.
- **Shiprocket Auth Persistence**: Created the `ShiprocketAuth` model to securely store the background JWT token.

## 2. Authentication & 9-Day Automation
- **`lib/shiprocket-auth.ts`**: Handles the Shiprocket login protocol. It checks if the stored token is valid or expiring soon (6-hour buffer) and refreshes it as needed.
- **`app/api/cron/shiprocket-auth/route.ts`**: A dedicated API endpoint designed for a cron job to trigger the token refresh every 9 days, ensuring uninterrupted connectivity.

## 3. Fulfillment Orchestration
- **`lib/shiprocket.ts`**: Low-level utility for `createOrder`, `generateAWB`, and `requestPickup` API calls.
- **`lib/services/fulfillment.ts`**: A coordination service that fetches the order, formats the payload for Shiprocket, and executes the sequence:
  1. Create Shiprocket Order
  2. Generate AWB
  3. Request Pickup
  4. Update Database Tracking IDs

## 4. Checkout Sync
- **Server Actions**: Updated `createOrder` and `payment/callback` to trigger the fulfillment service automatically.
  - **Cash on Delivery**: Fulfillment triggers immediately upon order creation.
  - **Online Payment**: Fulfillment triggers only after a successful PhonePe settlement handshake.

## 5. Tracking & Webhooks
- **`app/api/webhooks/shiprocket/route.ts`**: A webhook listener for Shiprocket tracking notifications. It synchronizes real-time status updates (e.g., "Shipped", "Delivered") back to the local database.
- **UI Updates**:
  - Added a **Logistics Telemetry** section to the Order Detail page (`/account/orders/[id]`).
  - Integrated a **Live Tracking Portal** link for customers.
  - Completely overhauled the Account and Order Detail pages to the new **Light Theme** for consistency.

## 6. Verification Steps Taken
- [x] Prisma client regenerated with new fields.
- [x] Env variables for Shiprocket configured in `.env.local`.
- [x] Checkout UI updated with Phone Number field.
- [x] Fulfillment service tested for data prep logic.
