# Phase 6: Logistics & Post-Purchase Automation (📦 Shiprocket)

## Objective
Automate the order fulfillment process and provide live tracking to customers using Shiprocket integration.

## Wave 1: Data Model & Auth Persistence
**Goal**: Prepare the database for Shiprocket credentials and order tracking.

- [ ] Task 1.1: Update Prisma schema with Shiprocket fields.
- [ ] Task 1.2: Add `ShiprocketAuth` model for JWT token persistence.
- [ ] Task 1.3: Run migrations and regenerate Prisma client.

## Wave 2: Auth Automation & Cron
**Goal**: Ensure a valid Shiprocket JWT is always available.

- [ ] Task 2.1: Implement `lib/shiprocket-auth.ts` for login logic.
- [ ] Task 2.2: Implement `app/api/cron/shiprocket-auth/route.ts` for 9-day refresh.
- [ ] Task 2.3: Verify auth persistence in database.

## Wave 3: Shiprocket Utility Service
**Goal**: Build the core API integration for fulfillment.

- [ ] Task 3.1: Implement `lib/shiprocket.ts` with token retrieval and base API wrappers.
- [ ] Task 3.2: Implement `createOrder`, `generateAWB`, and `requestPickup` functions.
- [ ] Task 3.3: Manual verification of API calls (Mock or Sandbox).

## Wave 4: Integration & Webhooks
**Goal**: Connect checkout to fulfillment and tracking.

- [ ] Task 4.1: Update `createOrder` action or checkout flow to trigger fulfillment.
- [ ] Task 4.2: Create `app/api/webhooks/shiprocket/route.ts` for tracking updates.
- [ ] Task 4.3: Map tracking updates to user account order view.

## Verification Protocol
1) `npx prisma validate`
2) Manual trigger of cron route.
3) Verification of shipment creation in Shiprocket dashboard (if sandbox available).
4) Simulated webhook POST and verification of DB update.
