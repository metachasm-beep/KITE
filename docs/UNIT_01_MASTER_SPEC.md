# UNIT_01 // MASTER_SPEC v1.0

## 0. EXECUTIVE VISION
UNIT_01 is a physical design protocol. We define the architecture of geometric form through high-density material studies. We do not sell "products"; we release "units" that serve as tactile anchors in a digital-first world.

---

## 1. THE PROTOCOL (Design Rules)
Every interface and physical object must adhere to the three pillars:

- **1.1. Material Logic**: Aesthetic decisions must feel derived from engineering requirements. 
- **1.2. HUD Continuity**: The digital interface is a telemetry layer for the physical archive. 
- **1.3. Minimal Friction**: No decorative clutter. Only technical necessity.

---

## 2. SYSTEM ARCHITECTURE (Route Map)

### 2.1. Storefront System (Desire)
- `/`: **SYSTEM_CORE** (Hero / Brand Ethos / Top Studies)
- `/collections`: **THE_ARCHIVE** (Index of all released units)
- `/collections/[slug]`: **UNIT_SPEC** (Technical breakdown of a single study)
- `/about`: **PROTOCOL_LOG** (Philosophy and manufacturing narrative)

### 2.2. Commerce System (Acquisition)
- `/cart`: **BUFFER_ALLOCATION** (Temporary queue for acquisition)
- `/checkout`: **INIT_ACQUISITION** (Transaction and logistics decryption)

### 2.3. Account System (Identity)
- `/account`: **MISSION_CONTROL** (Collector dashboard / Unit ownership)
- `/account/orders`: **LOGISTICS_HISTORY** (Detailed history of all acquisitions)
- `/account/orders/[id]`: **TRACKING_TELEMETRY** (Live shipping data)

---

## 3. DATA MODEL (Schema Alignment)

### 3.1. Artifact (Unit)
- `slug`: Unique identifier (e.g., `void-figure-01`)
- `title`: Upper-case technical name.
- `series`: The logic group (e.g., `SERIES 01 // ORIGIN`).
- `price`: Retail value in local currency.
- `status`: `AVAILABLE` (Localized) or `SOLD_OUT` (De-fragmented).

### 3.2. Specs (Telemetry)
- Each unit must have at least 4 technical specs (Material, Finish, Dimensions, Weight).
